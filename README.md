# KSC Settlement Refiner

경신그룹 해외법인 결산 데이터 통합 및 분석 도구

## 프로젝트 개요

5개 해외법인(KSCP, KCTR, KSCCZ, KSCE, KSCI)의 결산 데이터와 사업계획 데이터를 Excel 파일에서 자동으로 추출하여 통합 DB를 생성하는 Python 기반 엔진입니다.

### 주요 기능

- Excel 파일 자동 탐지 및 파싱 (`.xlsx`, `.xlsm`)
- 법인별 맞춤형 파서 (시트 구조 자동 인식)
- 환율 자동 적용 (현지통화 → KRW 변환)
- 통합 DB 생성 (3개 시트: 통합_DB, 법인별_요약, 경영보고서)
- 실적/계획 데이터 구분 관리

## 지원 파일 형식

### KSCP (폴란드)
**결산 파일:**
- 시트명: `결산 종합`, `월종합`, `보고용자료` 중 하나
- 통화: KRW

**사업계획 파일:**
- 시트명: `사업계획`, `KSCP 계획` 중 하나
- 통화: KRW

### KCTR (터키)
**결산 파일:**
- 시트명: `★PL(Report)`, `KCTR 실적`, `PL` 중 하나
- 통화: TRY (터키 리라)
- 특징: 누적 실적 형식 지원, 영문 계정과목 인식

**사업계획 파일:**
- 시트명: `Summary`
- 통화: KRW (환산 후)

### KSCCZ (체코)
**결산 파일:**
- 시트명: `★손익계산서` 또는 `손익계획`
- 통화: 체코 코루나 (CZK)

**사업계획 파일:**
- 시트명: `손익계획`, `제조원가계획`
- 통화: 원화 환산

### KSCE (세르비아)
**결산 파일:**
- 시트명: `PL(KR)`, `PL(Report)`, `PL2026(KR)`, `KSCE 실적`, `Summary`, `손익계산서`, `PL` 중 하나
- 통화: RSD (세르비아 디나르)

**사업계획 파일:**
- 시트명: 결산과 동일
- 통화: RSD

### KSCI (인도)
**결산 파일:**
- 시트명: 손익계산서 관련 시트
- 통화: INR (인도 루피)

**사업계획 파일:**
- 시트명: 사업계획 관련 시트
- 통화: INR

## 설치 및 실행

### 요구사항
- Python 3.10 이상
- openpyxl 라이브러리

### 설치
```bash
pip install openpyxl
```

### 실행
```bash
python engine.py [입력_디렉토리] [연도]

# 예시
python engine.py /home/user/data 2026
```

### 출력
- 위치: `./output/consolidated_db_YYYYMMDD_HHMMSS.xlsx`
- 포함 시트:
  1. **통합_DB**: 전체 데이터 (귀속연월, 법인코드, 데이터타입, 대분류, 중분류, 계정과목, 현지통화, 현지금액, 적용환율, KRW금액)
  2. **법인별_요약**: 법인별/기간별 주요 계정과목 집계
  3. **경영보고서**: PL/제조원가/이익률 분석 (실적당월, 계획당월, 실적누계, 계획누계)

## 버전 관리 규칙

이 프로젝트는 수정 사항이 발생할 때마다 **버전 번호를 기반으로 새로운 폴더와 프로젝트 이름**을 생성하여 버전을 관리합니다.

초기 버전은 **v1.0**으로 시작합니다.

### 1. 마이너(Minor) 수정
- **조건:** 버그 수정, 작은 기능 추가 등 기존 아키텍처를 크게 변경하지 않는 수정
- **버전 변경 규칙:** `v1.1` ➔ `v1.2`
- **폴더/프로젝트 명명 규칙:** `프로젝트명_v1.2` (기존 폴더를 복사하여 버전에 맞게 폴더명과 프로젝트명을 변경합니다)

### 2. 메이저(Major) 수정
- **조건:** 대규모 아키텍처 변경, 전체적인 리팩토링, 핵심 기능의 전면적인 개편 등
- **버전 변경 규칙:** `v2.0` ➔ `v3.0`
- **폴더/프로젝트 명명 규칙:** `프로젝트명_v3.0` (기존 폴더를 기반으로 새로운 메이저 버전 폴더명과 프로젝트명을 생성하여 작업합니다)

## 작업 흐름 (Workflow)
1. 새로운 수정 사항이 발생하면 기존 버전의 폴더를 복사합니다. (예: `MyProject_v1.0` ➔ 복사)
2. 복사된 폴더의 이름을 업데이트할 버전에 맞게 변경합니다. (예: `MyProject_v1.1` 또는 `MyProject_v2.0`)
3. 폴더 내 프로젝트 설정 파일(예: `.csproj`, `package.json`, `pom.xml` 등)에서 프로젝트 이름을 새 버전에 맞게 수정합니다.
4. 새로 생성한 폴더 환경에서 코드 수정 및 빌드 작업을 진행합니다.
5. 각 버전별 폴더를 통해 언제든 이전 버전의 구동 상태를 확인하거나 롤백할 수 있습니다.
